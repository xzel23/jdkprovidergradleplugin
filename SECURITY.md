# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it privately.

**Please do not open a public issue for security-related problems.**

### How to report

1. **GitHub Private Vulnerability Reporting**: The preferred way to report a vulnerability is through
   GitHub's [Private Vulnerability Reporting](https://docs.github.com/en/code-security/how-tos/report-and-fix-vulnerabilities/privately-reporting-a-security-vulnerability)
   feature. This allows you to report the issue directly to the maintainers in a secure and private manner.
2. **Email**: Alternatively, you can contact the maintainer directly via email at [axh@dua3.com](mailto:axh@dua3.com).
   Please use a descriptive subject line (e.g., "Security Vulnerability Report: [Project Name]").

### What to include

To help us address the issue quickly, please include as much information as possible:

* A detailed description of the vulnerability.
* Steps to reproduce the issue (including a minimal project if possible).
* Potential impact of the vulnerability.
* Any suggested mitigations or fixes.

Thank you for helping to keep this project secure!
